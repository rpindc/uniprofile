Design system reference

A quiet, precise set.

Twenty-four icons across three categories. Single 1.25px stroke. Sized to read at fourteen pixels and scale up cleanly to sixty-four. No fills except for status semantics.

Stroke 1.25px
Cap round
Corner 1px radius
Grid 24×24
Color currentColor
Padding 2px inset
Optical size 16px target
Status fill semantic only

How to use

For implementers

Include uniprofile_icons.svg once in your app shell, then reference any icon with a <use> tag. Color is inherited from the parent's CSS color property.

Passport ready
<svg class="up-icon color-ready"><use href="#icon-ready"/></svg>
Action needed
<svg class="up-icon color-attention"><use href="#icon-attention"/></svg>
Passport
<svg class="up-icon"><use href="#icon-passport"/></svg>

Recommended CSS

/* Base icon class */ .up-icon { width: 1em; height: 1em; flex-shrink: 0; display: inline-block; vertical-align: middle; } /* Status color helpers */ .color-ready { color: #2d6a4f; } .color-attention { color: #ba7517; } .color-critical { color: #c44d4d; } .color-pending { color: #8a8a86; } .color-standby { color: #b8b8b3; }

Rules to hold

  1. Status icons get semantic color. Action and Object icons stay in ink unless explicitly overridden.
  2. Never modify a symbol definition. If you need a variant, create a new symbol.
  3. Never mix this set with other icon libraries. Lucide, Heroicons, Material — none of them. The visual system holds because every icon was drawn to the same rules.
  4. When in doubt, use Document. If a generic file or attachment icon is needed, use icon-document. Don't introduce new generic glyphs.
